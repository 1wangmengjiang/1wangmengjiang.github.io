    git冲突问题

    团队协作中，可能两个人同时编辑了同一段代码，可能会有冲突：

    解决：

    1.先解决冲突问题，编辑好代码

    2.修改完冲突之后：

    ​ B： git add .

    ​ B: git commit -m ‘修改冲突’

    ​ B: git push origin dev

    ​ A: 在A分支里面 git pull origin dev

    ​ B: 在B分支里面 git pull origin dev

    避免提交一些不必要的文件

    1. 清空add之前的缓存： git rm -r –cached .

    2. 创建一个文件，文件名：.gitignore

    3. 在文件内写入不需要传入的文件名称

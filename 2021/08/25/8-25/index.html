怎么解决跨域？

  1. webapck-dev-server代理

    缺点：生产环境无效

  2. cors √

    优点：后端通用的方案，通用性强

    缺点：二次请求【①options：探测有没有跨域 ②真正的请求 put】

  3. nginx 反向代理 √

    优点：反向代理linux

  4. jsonp

    缺点：只能get请求，模拟script标签

  5. H5 domain

    优点：cookie

CORS使用

  1. server服务器加载
    npm i express-generator -g
    express server -e

  2. server>app.js配置COR 【注意：要在请求之前配置！！】

  1. server>bin>www
  1. serve>routes 下创建自己的router.js—-编写方法

  1. server>app.js

  2. 封装axios

    ① 【utils>request.js】

    ②【api>students.js】

  3. 在所需文件里面引入使用

  4. server与app必须同时运行

    每次更新都自己刷新，不需要重新启动
    下载：
    一、npm i nodemon -s
    二、package.json中配置
    “scripts”: {

    "start": "node ./bin/www",
"auto":"nodemon ./bin/www"

    },
    三、启动：npm run auto
